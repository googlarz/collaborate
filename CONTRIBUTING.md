# Contributing to collaborate

A Claude Code skill for collaborative document authoring. Contributions welcome.

## Before you start

- Check [open issues](https://github.com/googlarz/collaborate/issues) and [discussions](https://github.com/googlarz/collaborate/discussions)
- For changes to the skill workflow, open a discussion first — process changes need consensus

## What to contribute

- **Workflow improvements** — better steps, clearer gates, tighter verification
- **Edge case handling** — scenarios the skill doesn't cover well
- **Examples** — real-world usage patterns in the README
- **Bug reports** — describe the prompt you used and what went wrong

## Submitting a PR

1. Fork → branch from `main`
2. Edit `SKILL.md` — follow the existing structure (Overview, When to Use, Process, Verification)
3. Keep the skill focused — if you're adding a new capability, consider whether it belongs here or in a separate skill
4. Open the PR with a concrete example of what your change improves

## Skill format

Every section must remain actionable. Avoid vague advice — every step should be something an agent can execute and verify. See the [agent-skills anatomy guide](https://github.com/addyosmani/agent-skills) for reference.
